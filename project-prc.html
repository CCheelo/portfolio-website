PRC System — Player Registry Centre

2025–2026 · Frontend Project Manager JavaScript · Firebase · Bootstrap In active development

A web-based player registry centre designed to let academy representatives register and manage players, view profiles, and track competition data. Built collaboratively with a multi-team development structure using a feature-branch Git workflow.

Summary Overview

The Platform

PRC System is a multi-page web application for managing football academies and their players. The platform includes authentication, an academies dashboard, player listings, individual profile pages, and a data center for awards and competition outcomes.

Tech Stack

  • → Frontend — HTML, CSS, JavaScript, Bootstrap
  • → Backend — Firebase (authentication, Firestore database, storage)
  • → Data — JSON-structured player, academy, and award records

Team Structure

The project is split across two teams: Frontend (HTML, CSS, JavaScript, Bootstrap, and UI/UX design) and Backend (Firebase authentication, Firestore database, and storage). Both teams use a feature-branch Git workflow with PRs into dev, with a single project lead merging dev into main for stable releases.

My Role

Frontend Project Manager — leading the frontend team across all HTML, CSS, JavaScript, and UI/UX work. Responsibilities include coordinating task assignments, reviewing and merging frontend PRs, enforcing design consistency, and ensuring the frontend integrates cleanly with the backend team's Firebase implementation.

Also responsible for stakeholder engagement on behalf of the project — including presenting progress and demos to the company CEO and pitching the platform to potential football academies.

Status

The project is in active development. The live demo and screenshots will be added here once the platform reaches a stable milestone and any personal academy/player data has been replaced with anonymized examples.
